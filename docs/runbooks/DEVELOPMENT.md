# Development Runbook

Status: V1 local memory prototype

This runbook is the canonical path for setting up and checking Mneme locally. It keeps development bench-only: no ROS 2 runtime, sensors, actuators, GPIO, serial devices, firmware flashing, or robot hardware are involved.

## Prerequisites

- Python 3.11 or newer.
- Git.
- A shell from the repository root.

The package has one runtime dependency, `PyYAML`, and one development dependency, `pytest`.

## First-Time Setup

Create a virtual environment in the repository:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip setuptools wheel
python -m pip install -e '.[dev]'
```

The editable install makes `android_brain_memory` importable from the local `src/` tree and installs the test dependency.

## Canonical Local Check

Run the full local check:

```bash
python scripts/dev_check.py
```

The check runs, in order:

1. `python scripts/init_db.py`
2. `python scripts/smoke_test_memory.py`
3. `python -m pytest`

The scripts create or update `.local/android_brain_memory.sqlite3`. That file is local state and should not be committed.

## Individual Checks

Use these when debugging a specific layer:

```bash
python scripts/init_db.py
python scripts/smoke_test_memory.py
python -m pytest
```

If the package is not installed but dependencies are already available, run from the repository root with:

```bash
PYTHONPATH=src python3 scripts/dev_check.py
```

The recommended path is still the editable install because it matches CI more closely.

## CI Parity

GitHub Actions runs the same core commands on Python 3.11:

```bash
python -m pip install -e '.[dev]'
python scripts/init_db.py
python scripts/smoke_test_memory.py
python -m pytest
```

Before pushing meaningful changes, run `python scripts/dev_check.py` locally.

## Expected Local Artifacts

These paths are ignored by git:

- `.venv/`
- `.local/`
- `.pytest_cache/`
- `__pycache__/`
- `*.egg-info/`

Do not commit local databases, virtual environments, caches, or generated bytecode.

## Troubleshooting

If `python` is not found, use `python3` to create the virtual environment, then use `python` after activating `.venv`.

If `pytest` is missing, activate the virtual environment and rerun:

```bash
python -m pip install -e '.[dev]'
```

If a smoke script cannot import `android_brain_memory`, confirm the virtual environment is active and the editable install completed successfully.

If a database-related check behaves unexpectedly, remove `.local/android_brain_memory.sqlite3` and rerun:

```bash
python scripts/dev_check.py
```

Do not run hardware-facing commands as part of V1 memory development. The current project does not require connected robot hardware.
